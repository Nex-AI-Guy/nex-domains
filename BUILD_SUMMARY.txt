3. list

   - Includes DNS records
   - Shows check history

11. remove

DATABASE SCHEMA
===============

domains (18 fields):
  - id (PK, autoincrement)
  - domain (unique, indexed)
  - registrar (cloudflare, transip, other)
  - registration_date, expiry_date (indexed)
  - auto_renew (boolean)
  - nameservers (JSON)
  - ssl_expiry, ssl_issuer
  - dns_provider, hosting_provider
  - client (indexed)
  - monthly_cost
  - status (indexed): active, expired, parked, transferring
  - tags (JSON/CSV)
  - notes
  - last_checked
  - created_at, updated_at

dns_records (9 fields):
  - id (PK)
  - domain_id (FK, indexed)
  - record_type: A, AAAA, CNAME, MX, TXT, NS, SRV, CAA
  - name, content
  - ttl, proxied (boolean), priority
  - created_at, updated_at

domain_checks (5 fields):
  - id (PK)
  - domain_id (FK, indexed)
  - check_type: whois, dns, ssl, http
  - status, details (JSON)
  - checked_at (indexed)

DESIGN DECISIONS
================

1. No External Python Dependencies
   - Uses only Python stdlib
   - Reduces installation friction
   - subprocess for external tools

2. SQLite Database
   - Portable, no server required
   - Local-only data storage
   - Full-text search capability

3. Subprocess for External Tools
   - whois, dig, openssl as shell commands
   - Better separation of concerns
   - Easier to troubleshoot
   - Platform-specific tools work natively

4. Error Handling
   - Graceful failures with logging
   - Clear error messages to users
   - Timeouts on network operations
   - Recovery options documented

5. Design Patterns
   - Follows ClawHub skill conventions
   - Modular structure (config, storage, checkers)
   - Clear separation of concerns
   - Comprehensive documentation

6. Multi-Registrar Support
   - Agnostic registrar handling
   - Client/project organization
   - Cost tracking per domain
   - Status tracking for transfers
